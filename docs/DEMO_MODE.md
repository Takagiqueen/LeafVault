# LeafVault Demo 模式说明

Demo 模式是 LeafVault 面向公开体验场景提供的本地游客模式。它不需要注册、登录或邀请码，适合在公开部署页面中快速试用日记、账本、生活日历和本地备份等基础体验。

## 数据存在哪里

- Demo 数据只保存在当前浏览器的本地 IndexedDB。
- Demo 数据不会上传到服务器，也不会写入正式账号空间。
- Demo 使用独立工作区 `leafvault_demo_v1`，不会混入正式账号数据。
- Demo 使用前端本地标识 `demo-local-user`，不会伪造后端登录态。
- 清理浏览器站点数据、切换浏览器或更换设备后，Demo 数据可能消失。

## 可体验功能

Demo 模式允许使用：

- 写日记、编辑日记、删除日记。
- 添加本地图片体验，图片只保存在当前浏览器。
- 记录心情。
- 记账和查看统计。
- 查看生活日历。
- 本地加密导出和本地加密导入。
- PWA 添加到桌面、离线提示、空状态和加载状态体验。

## 需要正式账号的功能

以下功能在 Demo 模式下会被前端温和拦截，不会向服务器发送真实请求：

- 云端加密备份上传、下载、恢复和删除。
- 增量同步上传、远端变更下载、手动同步向导中的云端流程。
- 冲突云端下载。
- AI 润色。
- 修改用户名和修改密码。

统一提示为：

> Demo 模式仅支持本地体验。云端备份、多设备同步和账号设置需要正式账号。

## 图片处理边界

Demo 模式下的日记图片不会上传到正式 `uploads` 目录，也不会调用后端上传接口。前端会把图片转换为当前浏览器可保存的本地数据，仅用于体验图片预览和日记保存流程。

默认配置 `DEMO_SERVER_UPLOAD_ENABLED=false`，公开 Demo 不占用服务器上传空间。正式账号的图片上传仍会受到大小、类型和单篇数量限制；服务器会拒绝 SVG、超大图片和超过单篇上限的图片。

## 清空 Demo 数据

进入 Demo 后，个人中心会显示轻量状态卡。用户可以点击“清空数据”删除当前浏览器内的 Demo IndexedDB 数据。这个操作不会删除正式账号数据，也不会影响服务器数据库。

## 对服务器资源的影响

- Demo 不上传图片到服务器。
- Demo 不创建云端快照。
- Demo 不上传增量同步记录。
- Demo 不调用 AI API。

因此公开 Demo 不应占用正式 `uploads` 空间。便宜服务器仍需要定期备份 `data/` 和 `uploads/`，但 Demo 本身不会增加服务器上传文件。

## 隐私提醒

Demo 模式主要用于试用功能，不建议保存真实隐私内容。虽然数据不会上传服务器，但它仍然依赖当前浏览器的本地存储环境；共享设备、浏览器同步或系统清理工具都可能影响本地数据安全和持久性。

## 与正式账号的关系

- Demo 不会自动导入正式账号。
- 正式账号登录后不会自动读取 Demo 数据。
- 退出 Demo 后可以继续注册或登录正式账号。
- 正式账号才支持云端密文备份、多设备同步、账号设置和 AI 相关能力。
